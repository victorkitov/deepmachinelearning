Cвёрточные графовые сети

Cвёрточные графовые сети (graph convolutional networks) реализуют пересчёт исходных признаковых описаний xi\mathbf{x}_ixi​ соответствующих вершин viv_ivi​, учитывая контекст, то есть признаковые описания соседних вершин. Пересчёт производится много раз, начиная с hi0=xi\mathbf{h}^0_i=\mathbf{x}_ihi0​=xi​ и получая hi1,hi2,...hiK\mathbf{h}^1_i,\mathbf{h}^2_i,...\mathbf{h}^K_ihi1​,hi2​,...hiK​ для каждой вершины viv_ivi​.

Рассмотрим свёртку, действующую на однослойном изображении (чёрно-белом), в ней пересчёт производился по формуле

hik+1=f(∑j∈patch(i)wjhjk+b),h_{i}^{k+1}=f\left(\sum_{j\in\text{patch}(i)}w_{j}h_{j}^{k}+b\right),hik+1​=f​j∈patch(i)∑​wj​hjk​+b​,

где bbb - смещение, а {wj}j\{w_j\}_j{wj​}j​ - веса для соседних пикселей.

На графах используется только информация о соседних вершинах, причём нет понятий верхнего, нижнего, правого и левого соседей. Граф может быть повёрнут, но это будет тот же самый граф. Поэтому возможно только суммировать соседей с одним и тем же весом:

hik+1=f(w0zj+w1∑j∈N(i)hjk+b)h_{i}^{k+1}=f\left(w_{0}z_{j}+w_{1}\sum_{j\in\mathcal{N}(i)}h_{j}^{k}+b\right)hik+1​=f​w0​zj​+w1​j∈N(i)∑​hjk​+b​

Поскольку в графах каждой вершине сопоставляется не одно число, а вектор значений (эмбеддинг вершины), то пересчёт старых эмбеддингов hmk\mathbf{h}^k_mhmk​ в новые hmk+1\mathbf{h}^{k+1}_mhmk+1​ производится по формуле

znk=Aggregate({hmk}m∈N(n)),hnk+1=Update(hnk,znl),\begin{aligned}\mathbf{z}_{n}^{k} & =\text{Aggregate}\left(\left\{ \mathbf{h}_{m}^{k}\right\} _{m\in\mathcal{N}(n)}\right),\\ \mathbf{h}_{n}^{k+1} & =\text{Update}\left(\mathbf{h}_{n}^{k}, \mathbf{z}_{n}^{l}\right), \end{aligned} znk​hnk+1​​=Aggregate({hmk​}m∈N(n)​),=Update(hnk​,znl​),​

где

  • Aggregate(⋅)\text{Aggregate}\left(\cdot\right)Aggregate(⋅) - функция, агрегирующая информацию с соседних вершин,

  • Update(⋅)\text{Update}\left(\cdot\right)Update(⋅) - функция, обновляющая информацию о текущей вершине, используя её информацию с предыдущей итерации и агрегированной информации о её соседях.

Aggregate(⋅)\text{Aggregate}\left(\cdot\right)Aggregate(⋅) должна быть определена для любого числа соседей и не должна зависеть от порядка их обхода. Применение Argegate, а затем Update операции представляет собой одну итерацию алгоритма обмена сообщениями (message passing neural network [1]), который использует несколько итераций, а начальные значения для вершин определяются их начальными эмбеддингами: hn0=xn,n=1,2,...N.\mathbf{h}^0_n = \mathbf{x}_n, n=1,2,...N.hn0​=xn​,n=1,2,...N. Если xn\mathbf{x}_nxn​ не заданы, то их можно инициализировать one-hot закодированным значением числа соседей узла.

Алгоритм передачи сообщений представлен ниже:

ВХОД:

  • ненаправленный граф G=(V,E)G=(V,E)G=(V,E)

  • начальные эмбеддинги вершин {hn0}\{\mathbf{h}^0_n\}{hn0​}

  • функции Aggregate(⋅)\text{Aggregate}\left(\cdot\right)Aggregate(⋅) и Update(⋅)\text{Update}\left(\cdot\right)Update(⋅)

АЛГОРИТМ:

для k=1,2,...Kk=1,2,...Kk=1,2,...K:

  • znk=Aggregate({hmk}m∈N(n))\mathbf{z}_{n}^{k} =\text{Aggregate}\left(\left\{ \mathbf{h}_{m}^{k}\right\} _{m\in\mathcal{N}(n)}\right)znk​=Aggregate({hmk​}m∈N(n)​)

  • hnk+1=Update(hnk,znk)\mathbf{h}_{n}^{k+1} =\text{Update}\left(\mathbf{h}_{n}^{k}, \mathbf{z}_{n}^{k}\right)hnk+1​=Update(hnk​,znk​)

ВЫХОД:

  • итоговые эмбеддинги вершин hnK,  n=1,2,...N.\mathbf{h}^K_n,\; n=1,2,...N.hnK​,n=1,2,...N.

Применение KKK итераций алгоритма обмена сообщениями аналогично применению KKK свёрточных слоёв при обработке изображений. Функции Argegate и Update имеют свои настраиваемые параметры, причем в общем случае они свои на каждом слое (на каждой итерации алгоритма).

Обратим внимание, что каждая итерация алгоритма повышает область видимости (receptive field) результирующего эмбеддинга на один переход по графу. Это проиллюстрировано ниже для отдельной вершины.

Агрегирующая функция​

Функцию Aggregate можно определять по-разному, лишь бы она была определена для любого числа соседей и не зависела от их порядка. Самый простой вариант - через сумму эмбеддингов:

Aggregate(⋅)=∑m∈N(n)hmk\text{Aggregate}\left(\cdot\right)=\sum_{m\in\mathcal{N}(n)}\mathbf{h}_{m}^{k}Aggregate(⋅)=m∈N(n)∑​hmk​

Также можно использовать усреднение:

Aggregate(⋅)=1∣N(n)∣∑m∈N(n)hmk\text{Aggregate}\left(\cdot\right)=\frac{1}{\left|\mathcal{N}(n)\right|}\sum_{m\in\mathcal{N}(n)}\mathbf{h}_{m}^{k}Aggregate(⋅)=∣N(n)∣1​m∈N(n)∑​hmk​

Усреднение работает более устойчиво на вершинах, содержащих сильно различающееся число соседей, но, в отличие от суммы, не способно моделировать зависимость от числа соседей, поскольку при усреднении эта информация теряется.

Также можно усреднять, учитывая число соседних узлов у каждого соседа [2]:

Aggregate(⋅)=1∣N(n)∣⋅∣N(m)∣∑m∈N(n)hmk\text{Aggregate}\left(\cdot\right)=\frac{1}{\sqrt{\left|\mathcal{N}(n)\right|\cdot\left|\mathcal{N}(m)\right|}}\sum_{m\in\mathcal{N}(n)}\mathbf{h}_{m}^{k}Aggregate(⋅)=∣N(n)∣⋅∣N(m)∣​1​m∈N(n)∑​hmk​

Эмпирически при многократной агрегации на последних слоях алгоритма обмена сообщениями эмбеддинги получаются слишком сглаженными (over-smoothed) за счёт многократных усреднений. Предотвратить этот эффект можно, используя в качестве агрегации поэлементного максимума:

Aggregate(⋅)=maximum({hmk}m∈N(n))\text{Aggregate}\left(\cdot\right)=\text{maximum}\left(\left\{ \mathbf{h}_{m}^{k}\right\} _{m\in\mathcal{N}(n)}\right)Aggregate(⋅)=maximum({hmk​}m∈N(n)​)

Наиболее общим видом усреднения, способным моделировать любую дифференцируемую зависимость, инвариантную к перенумерации вершин, является использование двух многослойных персептронов (multi-layer perceptron, MLP) со своими настраиваемыми параметрами w\mathbf{w}w и v\mathbf{v}v [3]:

Aggregate(⋅)=MLPw(∑m∈N(n)MLPv(hmk))\text{Aggregate}\left(\cdot\right)=\text{MLP}_{\mathbf{w}}\left(\sum_{m\in\mathcal{N}(n)}\text{MLP}_{\mathbf{v}}\left(\mathbf{h}_{m}^{k}\right)\right)Aggregate(⋅)=MLPw​​m∈N(n)∑​MLPv​(hmk​)​

В последней формуле вместо суммы можно использовать другие перечисленные выше виды агрегации.

Агрегация со вниманием​

Можно использовать механизм внимания (attention mechanism) при агрегации узлов [4]. Для этого

  1. вычисляется внимание, с которым узел vnv_nvn​ должен смотреть на каждого своего соседа vmv_mvm​, используя некоторую функцию snm=g(hnk,hmk)s_{nm}=g(\mathbf{h}^k_n,\mathbf{h}^k_m)snm​=g(hnk​,hmk​);

  2. коэффициенты внимания пропускаются через SoftMax преобразование:

    {anm}m∈N(n)=SoftMax({snm}m∈N(n))\left\{ a_{nm}\right\}_{m\in\mathcal{N}(n)}=\text{SoftMax}\left(\left\{ s_{nm}\right\}_{m\in\mathcal{N}(n)}\right){anm​}m∈N(n)​=SoftMax({snm​}m∈N(n)​)

  3. Агрегация информации с узлов осуществляется пропорционально найденному вниманию:

    znk=∑n∈N(n)anmhmk\mathbf{z}_{n}^{k}=\sum_{n\in\mathcal{N}(n)}a_{nm}\mathbf{h}_{m}^{k}znk​=n∈N(n)∑​anm​hmk​

Например, внимание может вычисляться через билинейную форму с настраиваемой матрицей W∈RD×DW\in\mathbb{R}^{D\times D}W∈RD×D:

amn=exp(hnTWhm)∑m′∈N(n)exp(hnTWhm′)a_{mn}=\frac{\text{exp}\left(\mathbf{h}_{n}^{T}W\mathbf{h}_{m}\right)}{\sum_{m'\in\mathcal{N}(n)}\text{exp}\left(\mathbf{h}_{n}^{T}W\mathbf{h}_{m'}\right)}amn​=∑m′∈N(n)​exp(hnT​Whm′​)exp(hnT​Whm​)​

В более общем случае внимание может вычисляться через многослойный персептрон:

amn=exp(MLP(hn,hm))∑m′∈N(n)exp(MLP(hn,hm′))a_{mn}=\frac{\text{exp}\left(\text{MLP}\left(\mathbf{h}_{n},\mathbf{h}_{m}\right)\right)}{\sum_{m'\in\mathcal{N}(n)}\text{exp}\left(\text{MLP}\left(\mathbf{h}_{n},\mathbf{h}_{m'}\right)\right)}amn​=∑m′∈N(n)​exp(MLP(hn​,hm′​))exp(MLP(hn​,hm​))​

Аналогично тому, как это сделано в трансформере, можно применять несколько головок внимания со своими весами, конкатенировать полученные результаты и приводить результат к исходному DDD-мерному распределению, используя линейный слой.

Обновляющая функция​

Типичным способом определения функции Update является использование одного линейного слоя:

hnk+1=Update(hnk,znk)=f(Whk+Vznk+b),\mathbf{h}^{k+1}_n=\text{Update}\left(\mathbf{h}_{n}^{k},\mathbf{z}_{n}^{k}\right)=f\left(W\mathbf{h}^{k}+V\mathbf{z}_{n}^{k}+\mathbf{b}\right),hnk+1​=Update(hnk​,znk​)=f(Whk+Vznk​+b),

где f(⋅)f(\cdot)f(⋅) - функция нелинейности, применяемая поэлементно. Популярное способ передачи сообщений - агрегировать суммированием и использовать одну и ту же матрицу весов для целевого узла и его соседей:

hnk+1=f(W∑m∈N(n)∪nhmk+b)\mathbf{h}_{n}^{k+1}=f\left(W\sum_{m\in\mathcal{N}(n)\cup n}\mathbf{h}_{m}^{k}+\mathbf{b}\right)hnk+1​=f​Wm∈N(n)∪n∑​hmk​+b​

Чтобы облегчить настройку параметров сети, а также частично избавиться от сглаживания эмбеддингов (когда они становятся слишком похожими за счёт повторного усреднения по ним) можно использовать в качестве выхода сумму преобразования с исходным входом, как в ResNet-блоках:

hnk+1=Update(hnk,znk)+hnk\mathbf{h}_{n}^{k+1}=\text{Update}\left(\mathbf{h}_{n}^{k},\mathbf{z}_{n}^{k}\right)+\mathbf{h}_{n}^{k}hnk+1​=Update(hnk​,znk​)+hnk​

Альтернативно можно в качестве итогового представления каждой вершины использовать конкатенацию её представлений на всех итерациях передачи сообщений:

hnfinal=[hn1;hn1;...hnK]\mathbf{h}^{\text{final}}_n = [\mathbf{h}^1_n; \mathbf{h}^1_n; ... \mathbf{h}^K_n]hnfinal​=[hn1​;hn1​;...hnK​]

Тогда, даже если появится сглаживание на последних этапах, эмбеддинги вершин останутся различимыми за счёт учёта более ранних эмбеддингов.

Инвариантность к перенумерациям​

В матричном виде операции Aggregate и Update можно описать цепочкой преобразований

H1=F(X,A,W1)H2=F(H1,A,W2)⋯HK=F(HK,A,WK)\begin{aligned}H^{1} & =F\left(X,A,W^{1}\right)\\ H^{2} & =F\left(H^{1},A,W^{2}\right)\\ & \cdots\\ H^{K} & =F\left(H^{K},A,W^{K}\right) \end{aligned}H1H2HK​=F(X,A,W1)=F(H1,A,W2)⋯=F(HK,A,WK)​

где AAA - матрица смежности вершин, Hi=[h1i,...hNi]∈RD×NH^i=[h^i_1,...h^i_N]\in\mathbb{R}^{D\times N}Hi=[h1i​,...hNi​]∈RD×N - матрица, векторы которой являются внутренними представлениями узлов на iii-м шаге алгоритма обмена сообщениями, а WiW^iWi - матрица настраиваемых весов на iii-м шаге.

На каждом шаге алгоритма преобразование F(⋅)F(\cdot)F(⋅) должно обладать эквивариантностью (equivariance) к перенумерации узлов - если их задать в другом порядке, то его действие останется тем же, что математически записывается в виде

HkP=F(HkP,  PT⋅A⋅P,  Wk)H^{k}P = F\left(H^{k}P,\;P^T\cdot A\cdot P,\;W^{k}\right)HkP=F(HkP,PT⋅A⋅P,Wk)
Задача

Запишите в матричном виде описанные выше операции Aggregate & Update и покажите, что условие эквивариантности выполнено.

Учёт эмбеддингов рёбер и всего графа​

Алгоритм передачи сообщений можно обобщить, чтобы он обновлял не только эмбеддинги вершин, но также эмбеддинги рёбер и всего графа целиком:

ВХОД:

  • ненаправленный граф G=(V,E)G=(V,E)G=(V,E)

  • начальные эмбеддинги вершин {hn0}\{\mathbf{h}^0_n\}{hn0​}

  • начальные эмбеддинги рёбер {enm0}\{\mathbf{e}^0_{nm}\}{enm0​}

  • начальный эмбеддинг графа g0\mathbf{g}^0g0

  • функции Aggregate(⋅)\text{Aggregate}\left(\cdot\right)Aggregate(⋅) и Update(⋅)\text{Update}\left(\cdot\right)Update(⋅)

АЛГОРИТМ:

для k=1,2,...Kk=1,2,...Kk=1,2,...K:

  • enmk+1=Updateedge(enmk,hnk,hmk,gk)\mathbf{e}_{nm}^{k+1}=\text{Update}_{\text{edge}}\left(\mathbf{e}_{nm}^{k},\mathbf{h}_{n}^{k},\mathbf{h}_{m}^{k},\mathbf{g}^{k}\right)enmk+1​=Updateedge​(enmk​,hnk​,hmk​,gk)

  • znk+1=Aggregatenode({enmk}m∈N(n))\mathbf{z}_{n}^{k+1}=\text{Aggregate}_{\text{node}}\left(\left\{ \mathbf{e}_{nm}^{k}\right\} _{m\in\mathcal{N}(n)}\right)znk+1​=Aggregatenode​({enmk​}m∈N(n)​)

  • hnmk+1=Updatenode(hnk+1,znk+1,gk)\mathbf{h}_{nm}^{k+1}=\text{Update}_{\text{node}}\left(\mathbf{h}_{n}^{k+1},\mathbf{z}_{n}^{k+1},\mathbf{g}^{k}\right)hnmk+1​=Updatenode​(hnk+1​,znk+1​,gk)

  • gnk+1=Update(gk,{hnk+1},{enmk+1})\mathbf{g}_{n}^{k+1}=\text{Update}\left(\mathbf{g}^{k},\left\{ \mathbf{h}_{n}^{k+1}\right\} ,\left\{ \mathbf{e}_{nm}^{k+1}\right\} \right)gnk+1​=Update(gk,{hnk+1​},{enmk+1​})

ВЫХОД:

  • итоговые эмбеддинги {hnK},{enmK},gK\left\{ \mathbf{h}_{n}^{K}\right\} ,\left\{ \mathbf{e}_{nm}^{K}\right\} ,\mathbf{g}^{K}{hnK​},{enmK​},gK

Как видим, на каждой итерации

  1. вначале обновляются эмбеддинги каждого ребра по эмбеддингам его вершин;

  2. затем обновляется эмбеддинг контекста каждой вершины по эмбеддингам связанных с ним рёбер;

  3. потом происходит обновление эмбеддинга вершины по её контексту;

  4. наконец обновляется эмбеддинг всего графа

Учёт эмбеддингов вершин позволяет решать задачи на вершинах (например, предсказывать тип связи), а эмбеддинг всего графа может учитываться при регрессии/классификации всего графа целиком.
