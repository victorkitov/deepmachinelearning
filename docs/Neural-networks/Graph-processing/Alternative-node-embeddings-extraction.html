Эмбеддинги на основе случайных блужданий

Существуют и другие способы генерации эмбеддингов для вершин графа, состоящего из NNN вершин, каждой из которых сопоставлен некоторый начальный DDD-мерный эмбеддинг, заданный случайно.

DeepWalk​

В методе DeepWalk [1] из каждой вершины KKK раз запускается процесс случайного блуждания длины MMM. Если трактовать каждую вершину, как слово, то в результате получается K⋅NK\cdot NK⋅N предложений длины MMM каждое.

Для уточнения эмбеддингов по их расположениям на графе в DeepWalk предлагается использовать метод SkipGram модели Word2Vec, в котором по каждому сгенерированному пути на графе скользит окно ширины 2W+12W+12W+1 символ, и, используя эмбеддинги вершин нужно настроить языковую модель, в которой по центральной вершины окна нужно с как можно более высокой вероятностью предсказать соседние (справа и слева) вершины окна.

Идеи расширения метода

Обратим внимание, что используя ту же методологию, можно строить эмбеддинги вершин графа используя C-BOW версию модели Word2Vec или другую языковую модель.

Также начальные значения эмбеддингов можно инициализировать не случайно, а используя известные признаки каждого узла на графе.

Выходом алгоритма DeepWalk являются настроенные из языковой модели эмбеддинги для каждой вершины, которые по построению будут учитывать расположение вершины на графе - близким вершинам будут соответствовать более близкие эмбеддинги (их скалярное произведение будет больше).

Сравнение DeepWalk и алгоритма передачи сообщений

Метод DeepWalk более экономичен по памяти, чем алгоритм передачи сообщений (message passing algorithm) в классических графовых нейросетях, поскольку передача сообщений требует аккумуляции информации со всех соседей каждой вершины, в то время как настройка DeepWalk предполагает переходы лишь по одной случайно выбираемой каждый раз вершине.

С другой стороны, выбирая каждый раз лишь одну из соседних вершин, методу DeepWalk потребуется больше итераций до сходимости, поскольку лишь многократные обходы вершин смогут собрать информацию о всех их соседях.

Node2Vec​

Модель Node2Vec [2] также предназначена для построения эмбеддингов узлов графа. Она использует тот же механизм построения эмбеддингов, что и DeepWalk:

  • каждому узлу графа сопоставляется случайный вектор эмбеддингов;

  • из каждого узла генерируются случайные блуждания (random walks) по соседним узлам;

  • в сгенерированных блужданиях перебираются всевозможные последовательности узлов фиксированной длины (окна наблюдений);

  • в каждом окне наблюдений модель по эмбеддингу центрального узла в окне предсказывает вероятность появления всех прочих узлов в том же окне, как в модели SkipGram; пытаясь максимизировать эту вероятность, делается один шаг градиентного подъёма, уточняющий эмбеддинги задействованных узлов.

Ключевой идеей Node2Vec является наблюдение, что обход вершин графа из заданной вершины может быть устроен по-разному:

  • поиск в ширину (breadth-first search, BFS), при котором обход осуществляется в первую очередь по вершинам в окрестности начальной вершины;

  • поиск в глубину (depth-first search, DFS), при котором обход осуществляется таким образом, чтобы уходить максимально далеко от первоначальной вершины.

Эти две стратегии проиллюстрированы ниже красным и синим цветом [2]:

Node2Vec использует случайное блуждание 2-го порядка (2-nd order random walk) с гиперпараметрами p,q>0p,q>0p,q>0, управляющими характером обхода вершин, больше склоняющимся к обходу в глубину или обходу в ширину.

В частности, если только осуществился переход из вершины ttt в вершину vvv, то вероятность перехода из вершины vvv в новую вершину xxx считается по формуле

p(v→x∣v,t)={γ⋅1pесли x=t,γ⋅1если d(t,x)=1,γ⋅1qесли d(t,x)=2,p(v\to x|v,t)=\begin{cases} \gamma \cdot \frac{1}{p} & \text{если }x=t,\\ \gamma \cdot 1 & \text{если }d\left(t,x\right)=1,\\ \gamma \cdot \frac{1}{q} & \text{если }d\left(t,x\right)=2, \end{cases}p(v→x∣v,t)=⎩⎨⎧​γ⋅p1​γ⋅1γ⋅q1​​если x=t,если d(t,x)=1,если d(t,x)=2,​

где d(t,x)d(t,x)d(t,x) - длина минимального пути от вершины ttt в вершину xxx, а γ\gammaγ - взвешивающий коэффициент, нормализующий все вероятности суммироваться в единицу. Пример вероятностей перехода приведён ниже [2]:

Если связи на графе взвешенные (каждому ребру сопоставлено число, обозначающее силу связи), то вероятности переходов нужно ещё дополнительно домножить на веса соответствующих связей.

При малых значениях ppp обход будет вестись скорее в ширину, поскольку будут частые возвращения в исходную вершину. Малые значения qqq, наоборот, стимулируют поиск в глубину.

Обход в глубину будет выделять сообщества - у группы узлов, сильно связанных друг с другом напрямую или опосредованно, будут схожие эмбеддинги.

Обход в ширину будет определять структурные роли вершин (structural roles of nodes), т.е. является ли вершина

  • хабом (центральным узлом в сообществе);

  • связующим узлом (связывает несколько сообществ между собой);

  • периферийной вершиной (находящейся на краю сообщества).

Использование автокодировщика​

Каждую вершину графа можно охарактеризовать тем, с какими другими вершинами она связана, что задаётся соответствующей строкой автокодировщика.

Поэтому альтернативным подходом для извлечения эмбеддингов узлов графа является обучение автокодировщика восстанавливать строки матрицы смежности (adjency matrix). Тогда эмбеддингом вершины будет внутреннее представление посередине автокодировщика, через который пропущена соответствующая строка матрицы смежности.

Такая архитектура способна сразу выдавать эмбеддинг для новой вершины, но важно, чтобы автокодировщик содержал немного параметров, иначе он переобучится.

Такой подход используется, например, в статье [3], в которой также предложены дополнительные способы регуляризации для борьбы с переобучением.
