Решение задач на графах

Построение эмбеддингов вершин​

Вначале каждой вершине графа сопоставлен свой вектор данных:

vi→hi∈RD,i=1,2,...N.v_i \to \mathbf{h}_i \in \mathbb{R}^{D}, \quad i=1,2,...N.vi​→hi​∈RD,i=1,2,...N.

Этим вектором данных могут выступать начальные характеристики вершины. Например, если рассматривается граф социальной сети, в которой вершинами выступают люди, то hi\mathbf{h}_ihi​ может описывать пол, возраст и интересы, указанные в анкете. Если информации об узлах не предоставлено, можно задать её самостоятельно, передавая

  • one-hot закодированный номер вершины (для небольших графов),

  • геометрические характеристики вершины (такие как число соседей первого, второго порядка, минимальные длины путей до выбранных опорных узлов графа)

Далее стоит задача уточнить эти эмбеддинги так, чтобы учесть контекстную информацию о соседях (связанных вершинах) аналогично тому, как уточнялись эмбеддинги слов в архитектуре трансформера по контексту соседних слов. Для этого применяется итеративный алгоритм свёрточных графовых сетей (graph convolutional networks), описанный в следующей главе. На выходе алгоритма мы получаем уже эмбеддинги каждой вершины {hi1}i\{\mathbf{h}_i^1\}_i{hi1​}i​ с учётом контекста её соседей, {hi2}i\{\mathbf{h}_i^2\}_i{hi2​}i​ с учётом контекста соседей и соседей от соседей и т.д. После KKK итераций алгоритма получаем окончательные контекстные эмбеддинги {hi}i\{\mathbf{h}_i\}_i{hi​}i​, которые совместно будут храниться в матрице H=[h1,...hN]∈RD×NH=[\mathbf{h}_1,...\mathbf{h}_N]\in\mathbb{R}^{D\times N}H=[h1​,...hN​]∈RD×N.

Все контекстные эмбеддинги будут храниться в матрице H=[h1,...hN]∈RD×NH=[\mathbf{h}_1,...\mathbf{h}_N]\in\mathbb{R}^{D\times N}H=[h1​,...hN​]∈RD×N.

Задачи на графе целиком​

Рассмотрим задачи, в которых известно некоторое свойство для графов G1,...GMG_1,...G_MG1​,...GM​ и требуется предсказывать свойство для нового графа G=(V,E,X)G=(V,E,X)G=(V,E,X), содержащего NNN вершин. Поскольку обучающие и тестовые данные разделены, то это классическая задача обучения с учителем (supervised learning) или индуктивного обучения (inductive learning).

Корректно специфицированная модель должна быть инвариантной к перенумерации вершин (invariance to permutation), то есть результат не должен зависеть от того, как именно мы пронумеруем вершины графа. Формально это записывается как

y^(H)≡y^(H⋅P)для любой матрицы перестановки P.\hat{y}(H)\equiv \hat{y}(H\cdot P) \quad \text{для любой матрицы перестановки } P.y^​(H)≡y^​(H⋅P)для любой матрицы перестановки P.

Регрессия​

Для решения задачи регрессии на графе (например, определении температуры плавления материала) можно усреднить все контекстные эмбеддинги, а затем линейно преобразовать, чтобы предсказать целевую величину:

y^(G)=w0+wT⋅H⋅i/N,\hat{y}(G) = w_0+\mathbf{w}^T \cdot H \cdot \mathbf{i}/N,y^​(G)=w0​+wT⋅H⋅i/N,

где

  • i=[1,1,...1]T∈RD\mathbf{i}=[1,1,...1]^T\in\mathbb{R}^{D}i=[1,1,...1]T∈RD - вектор из единиц,

  • w0∈R,w∈RDw_0\in\mathbb{R}, \mathbf{w}\in\mathbb{R}^{D}w0​∈R,w∈RD - настраиваемые параметры.

Домножение на вектор из единиц и деление на число узлов NNN означает усреднение эмбеддингов.

Бинарная классификация​

Бинарную классификацию графа (например, классификацию того, является ли вещество, описанное графом, токсичным для человека) можно производить по формуле:

y^(G)=σ(w0+wT⋅H⋅i/N),\hat{y}(G) = \sigma(w_0+\mathbf{w}^T \cdot H \cdot \mathbf{i}/N),y^​(G)=σ(w0​+wT⋅H⋅i/N),

где σ(u)=1/(1+e−u)∈(0,1)\sigma(u)=1/(1+e^{-u})\in (0,1)σ(u)=1/(1+e−u)∈(0,1) - сигмоидная функция (сигмоида, sigmoid).

Многоклассовая классификация​

Многоклассовую классификацию графа (например, тип молекулы: углевод, белок, липид и т.д.) можно производить, вычислив CCC дискриминантных функций для каждого класса, а затем пропустить эти функции через SoftMax преобразование:

(p(y=1∣G)p(y=2∣G)⋯p(y=C∣G))=SoftMax(w0+W⋅H⋅i/N),\left(\begin{array}{c} p(y=1|G)\\ p(y=2|G)\\ \cdots\\ p(y=C|G) \end{array}\right)=\text{SoftMax}\left(\mathbf{w}_{0}+W\cdot H\cdot\mathbf{i}/N\right),​p(y=1∣G)p(y=2∣G)⋯p(y=C∣G)​​=SoftMax(w0​+W⋅H⋅i/N),

где настраиваемые параметры - это

  • w0∈RC\mathbf{w}_{0}\in \mathbb{R}^Cw0​∈RC - вектор смещений,

  • W∈RC×DW\in\mathbb{R}^{C\times D}W∈RC×D - матрица весов.

подсказка

Альтернативой усреднения эмбеддингов вершин графа могут выступать

  • суммирование эмбеддингов (в этом случае количество узлов графа будут влиять на результат, увеличивая разброс значений суммы)

  • поэлементный максимум эмбеддингов (в этом случае прогноз будет исходить не из средней или суммарной представленности каждого признака, а из наличия сильно выраженного признака хотя бы на одном из узлов; здесь можно провести аналогию между усредняющим и максимизирующим пулингом на изображениях)

Задача

Докажите, что модели в каждом случае обладают инвариантностью к перенумерации вершин. Для этого воспользуйтесь свойством P⋅i=i  ∀PP\cdot \mathbf{i}=\mathbf{i} \; \forall PP⋅i=i∀P (которое тоже докажите).

Обработка отдельных вершин​

Часто дан всего один граф, для каждой вершины которого нужно предсказать определённое свойство. Для части вершин это свойство известно, что используется для настройки параметров модели. Примером может выступать граф социальной сети, в котором узлы - это пользователи, а связи - отношения дружбы между ними.

Поскольку заранее известны вершины, для которых нужно что-то предсказать, которые к тому же связаны с обучающими вершинами (где свойство известно), то это задача частичного обучения (semi-supervised learning).

Корректно специфицированная модель должна обладать свойством эквивариантности к перенумерации вершин графа (equivariance to permutation), то есть при перенумерации вершин соответствующим образом должны перенумеровываться и выходы модели, чтобы сохранить связь с соответствующими вершинами.

Регрессия​

Рассмотрим задачу регрессии на каждой вершине графа. Например, для графа социальной сети нужно предсказать ожидаемое время, которое каждый пользователь проведёт в сети. Для вершины viv_ivi​ с контекстным эмбеддингом hi\mathbf{h}_ihi​ она решается по формуле

y^(hi)=w0+wThi.\widehat{y}(\mathbf{h}_{i})=w_{0}+\mathbf{w}^{T}\mathbf{h}_{i}.y​(hi​)=w0​+wThi​.

Для всех узлов прогноз строится как

y^(H)=w0+wTH.\widehat{y}(H)=w_{0}+\mathbf{w}^{T}H.y​(H)=w0​+wTH.

Бинарная классификация​

Рассмотрим задачу бинарной классификации каждой вершины графа. Например, для графа социальной сети нужно предсказать, является ли аккаунт реальным человеком или ботом. Для вершины viv_ivi​ с контекстным эмбеддингом hi\mathbf{h}_ihi​ она решается по формуле

p(y^=1∣hi)=σ(w0+wThi).p(\hat{y}=1|\mathbf{h}_{i}) = \sigma(w_{0}+\mathbf{w}^{T}\mathbf{h}_{i}).p(y^​=1∣hi​)=σ(w0​+wThi​).

Для всех узлов прогноз строится как

p(y^=1∣H)=σ(w0+wTH),\mathbf{p}(\hat{\mathbf{y}}=1|H) = \sigma(w_{0}+\mathbf{w}^{T}H),p(y^​=1∣H)=σ(w0​+wTH),

где сигмоида применяется поэлементно.

Многоклассовая классификация​

Рассмотрим задачу бинарной классификации каждой вершины графа на CCC классов. Например, для графа социальной сети нужно предсказать, является ли человек безработным, работающим по найму, самозанятым или предпринимателем. Для этого вычисляется CCC рейтингов каждого класса, которые преобразуются в набор вероятностей классов через SfotMax преобразование:

(p(y=1∣hi)p(y=2∣hi)⋯p(y=C∣hi))=SoftMax(w0+W⋅hi),\left(\begin{array}{c} p(y=1|\mathbf{h}_{i})\\ p(y=2|\mathbf{h}_{i})\\ \cdots\\ p(y=C|\mathbf{h}_{i}) \end{array}\right)=\text{SoftMax}\left(\mathbf{w}_{0}+W\cdot\mathbf{h}_{i}\right),​p(y=1∣hi​)p(y=2∣hi​)⋯p(y=C∣hi​)​​=SoftMax(w0​+W⋅hi​),

где настраиваемые параметры - это

  • w0∈RC\mathbf{w}_{0}\in \mathbb{R}^Cw0​∈RC - вектор смещений,

  • W∈RC×DW\in\mathbb{R}^{C\times D}W∈RC×D - матрица весов.

Сразу для всех вершин вероятности считаются как

P^(H)=SoftMax(w0+W⋅H),\hat{P}(H)=\text{SoftMax}\left(\mathbf{w}_{0}+W\cdot H\right),P^(H)=SoftMax(w0​+W⋅H),

где SoftMax применяется независимо к каждому столбцу.

Задача

Докажите, что все приведённые модели обладают эквивариантностью к перенумерации вершин графа.

Восстановление связей​

Рассмотрим один граф, часть вершин для которого известна, и надо восстановить пропущенные связи. Для этого можно посчитать вероятность связи между каждой парой узлов viv_ivi​ и vjv_jvj​ по формуле

p((vi,vj)-connected ∣ hi,hj)=σ(hiThj),p((v_i,v_j) \text{-connected}\,|\,\mathbf{h}_i,\mathbf{h}_j) = \sigma(\mathbf{h}_i^T\mathbf{h}_j),p((vi​,vj​)-connected∣hi​,hj​)=σ(hiT​hj​),

где σ(⋅)\sigma(\cdot)σ(⋅) - сигмоидная функция. По смыслу скалярное произведение вычисляет сочетаемость характеристик вершин друг с другом и соединяются вершины, обладающие схожими характеристиками. После расчёта вероятности связи для каждой пары вершин можно провести связи там, где эта вероятность больше некоторого порога ttt:

p((vi,vj)-connected ∣ hi,hj)>t  ⟹  connect vi,vj.p((v_i,v_j) \text{-connected}\,|\,\mathbf{h}_i,\mathbf{h}_j)>t \;\Longrightarrow\; \text{connect } v_i,v_j.p((vi​,vj​)-connected∣hi​,hj​)>t⟹connect vi​,vj​.

Гиперпараметр ttt управляет противоречием между точностью (precision) и полнотой (recall) восстановления связей и подбирается по качеству восстановления вершин на подмножестве вершин, на котором все связи известны, но были предварительно затёрты, чтобы на них не переобучиться.

Если эмбеддинги {hi}i\{\mathbf{h}_i\}_i{hi​}i​ настраиваются для решения других задач, то можно усложнить расчёт вероятности связи, введя дополнительную параметризацию:

p((vi,vj)-connected ∣ hi,hj)=σ(w0+hiTWhj),p((v_i,v_j) \text{-connected}\,|\,\mathbf{h}_i,\mathbf{h}_j) = \sigma(w_0+\mathbf{h}_i^T W \mathbf{h}_j),p((vi​,vj​)-connected∣hi​,hj​)=σ(w0​+hiT​Whj​),

где новые параметры - это

  • w0∈Rw_0\in\mathbb{R}w0​∈R - смещение,

  • W∈RD×DW\in\mathbb{R}^{D\times D}W∈RD×D - симметричная матрица весов, т.е. W=WTW=W^TW=WT.

Поскольку каждая симметричная матрица представима в виде разложения Холецкого

W=L⋅LT, L - нижнетреугольная матрицаW=L\cdot L^T, \text{ $L$ - нижнетреугольная матрица}W=L⋅LT, L - нижнетреугольная матрица

то перебирать всевозможные симметричные матрицы можно, настраивая матрицу LLL.

Представленная параметризация позволит перевести эмбеддинги узлов в новое пространство, более приспособленное для решения задачи восстановления рёбер.

Регрессия и классификация рёбер​

Можно решать задачи классификации и регрессии не узлов, а рёбер графа. Например в социальной сети предсказывать, какой будет характер взаимодействия между людьми и сколько сообщений они друг другу напишут.

Для этого нужно перевести исходный граф в граф рёбер (edge graph), превратив рёбра исходного графа в узлы. А далее применить те же техники классификации и регрессии, но для узлов графа рёбер.
