Локальное объяснение интерпретируемой моделью

Метод LIME (local interpretable model-agnostic explanations) позволяет объяснить прогноз сложной модели f(x) для интересуемого объекта x за счёт аппроксимации логики работы этой модели другой простой и интерпретируемой моделью в окрестности x. Например, в задаче кредитного скоринга нас может интересовать, почему для определённого клиента кредит не был одобрен. Прогноз возврата кредита осуществляется сложной и неинтерпретируемой нейросетью, но мы можем насэмплировать много клиентов, похожих на интересующего нас, посмотреть на прогнозы сложной модели для этих синтетических клиентов и настроить простую интерпретируемую модель (решающее дерево, линейная модель и др.) аппроксимирующию прогнозы сложной модели в окрестности интересуемого объекта. Простая модель обладает непосредственной интерпретацией, поэтому можно напрямую изучить её логику принятия решения, чтобы делать выводы о работе сложной модели в окрестности интересующего объекта. Алгоритм метода LIME устроен следующим образом:

  1. Выбрать xxx для которого нужно объяснить прогноз.

  2. Сгенерировать выборку локальные вариации x~1,x~2,...x~K\widetilde{x}_{1},\widetilde{x}_{2},...\widetilde{x}_{K}x1​,x2​,...xK​ относительно xxx

  3. Построить для вариаций прогнозы сложной моделью f(⋅)f\left(\cdot\right)f(⋅), получив выборку:

    {[x~1,f(x~1)]; [x~2,f(x~2)]; ...[x~K,f(x~K)]}\left\{ \left[\widetilde{x}_{1},f\left(\widetilde{x}_{1}\right)\right];~\left[\widetilde{x}_{2},f\left(\widetilde{x}_{2}\right)\right];~...\left[\widetilde{x}_{K},f\left(\widetilde{x}_{K}\right)\right]\right\}{[x1​,f(x1​)]; [x2​,f(x2​)]; ...[xK​,f(xK​)]}

  4. Взвесить объекты по близости к xxx:

    {[w1,x~1,f(x~1)]; [w2,x~2,f(x~2)]; ...[wK,x~K,f(x~K)]}\left\{ \left[w_{1},\widetilde{x}_{1},f\left(\widetilde{x}_{1}\right)\right];~\left[w_{2},\widetilde{x}_{2},f\left(\widetilde{x}_{2}\right)\right];~...\left[w_{K},\widetilde{x}_{K},f\left(\widetilde{x}_{K}\right)\right]\right\}{[w1​,x1​,f(x1​)]; [w2​,x2​,f(x2​)]; ...[wK​,xK​,f(xK​)]}

  5. Настроить интерпретируемую модель g(x)g\left(x\right)g(x) по взвешенной выборке (чем вес выше, тем сильнее объект учитывается).

  6. Исследовать интерпретируемую модель.

Для повышения объяснимости простой модели можно

  • в случае решающего дерева задать ограничение на его максимальную глубину

  • в случае линейной регрессии использовать L1L_{1}L1​ регуляризацию, чтобы контролировать число используемых признаков.

Число используемых признаков можно также сократить, используя процедуры отбора признаков forward-selection (последовательный выбор самых влияющих) или backward-selection (последовательное исключение самых незначимых). Также интерпретирующая модель может опираться на другие - более интерпретируемые признаки, отличные от исходных. При этом важно помнить, что простая интерпретируемая модель является лишь аппроксимацией сложной модели, поэтому важно предварительно оценить качество её аппроксимации. Например, для задачи регрессии это может быть ошибка аппроксимации относительно ошибки аппроксимации константным прогнозом:

ApproxError=1K∑k=1K(f(xk)−g(xk))21K∑k=1K(f(xk)−1K∑k=1Kf(xk))2\text{ApproxError}=\sqrt{\frac{\frac{1}{K}\sum_{k=1}^{K}\left(f\left(x_{k}\right)-g\left(x_{k}\right)\right)^{2}}{\frac{1}{K}\sum_{k=1}^{K}\left(f\left(x_{k}\right)-\frac{1}{K}\sum_{k=1}^{K}f\left(x_{k}\right)\right)^{2}}}ApproxError=K1​∑k=1K​(f(xk​)−K1​∑k=1K​f(xk​))2K1​∑k=1K​(f(xk​)−g(xk​))2​​

Если ошибка аппроксимации высока, нужно либо усложнить аппроксимирующую модель (увеличивая число признаков, ослабляя L_1 регуляризацию или увеличивая глубину дерева), либо уменьшить окрестность сэмплируемых вокруг x точек (которую при прочих равных лучше брать побольше).

Процесс сэмплирования объектов, похожих на заданный, зависит от характера объектов:

  • Для вектора вещественных чисел можно добавлять небольшой шум.

  • Для текста можно включать/исключать отдельные слова.

  • Для изображения: включать/исключать отдельные суперпиксели (superpixels) - области соседних пикселей, примерно похожих по цвету.
