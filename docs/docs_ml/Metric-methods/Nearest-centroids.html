Метод ближайших центроидов

Метод ближайших центроидов (nearest centroids) решает задачу классификации. Рассмотрим демонстрационные двумерные данные ниже, разбитые на три класса - синий, красный и зелёный.

Обучение метода заключается в вычислении центроидов для каждого класса.

μc=1Nc∑n:yn=cxn\mu_{c}=\frac{1}{N_{c}}\sum_{n: y_{n}=c}x_{n}μc​=Nc​1​n:yn​=c∑​xn​

Ниже они обозначены крестиками соответствующего цвета.

На этапе предсказания класса для объекта xxx вычисляются расстояния до каждого из центроидов. Объекту назначается тот класс, расстояние до которого меньше всего:

y^(x)=arg min⁡yρ(x,μy)\hat{y}(x)=\argmin_{y}\rho(x,\mu_{y})y^​(x)=yargmin​ρ(x,μy​)
Чему равны дискриминантные функции для этого метода?

Дискриминантная функция каждого из классов вычисляет рейтинг соответствующего класса - чем выше, тем класс более вероятен. Поэтому дискриминантная функция класса y будет равна расстоянию от x до центроида соответствующего класса со знаком минус (или любой другой убывающей функции от расстояния).

Всегда ли границы между классами будут линейными гиперплоскостями?

Границы между классами i и j задаются условием {x:gi(x)=gj(x)}\{x: g_i(x)=g_j(x)\}{x:gi​(x)=gj​(x)}, что в случае метода ближайших центроидов будет {x:ρ(x,μi)=ρ(x,μj)}\{x: \rho(x,\mu_i)=\rho(x,\mu_j)\}{x:ρ(x,μi​)=ρ(x,μj​)}. Это всегда будет линейной гиперплоскостью, если использовать Евклидову функцию расстояния.

Строго говоря, метод не является метрическим, поскольку оперирует расстояниями не до самих обучающих объектов, а до их средних. Однако его можно сделать метрическим, если в качестве центроидов выбирать не усредненные объекты по классу, а центральные объекты класса из обучающей выборки (т.е. такие объекты выборки, что расстояние от них до всех других объектов того же класса будет наименьшим).

За счет того, что сравнение xxx происходит лишь с центроидами, а не со всеми объектами выборки, прогноз будет строиться быстро. Однако метод способен выделять лишь выпуклые формы кластеров примерно сферической формы, поэтому в общем случае будет работать неточно. Его можно использовать как бейзлайн (простой метод задающий нижнюю границу на точность), а также усложнить, добавив дополнительные центроиды, характеризующие каждый класс.

Метод ближайших центроидов можно обобщить, используя разные функции расстояния, не обязательно Евклидову.
