Метод один-против-всех

Метод один-против-всех (one-vs-all, one-vs-rest) позволяет решать задачу многоклассовой классификации с помощью набора бинарных классификаторов.

Для этого для каждого класса c=1,2,...Cc=1,2,...Cc=1,2,...C обучим свой бинарный классификатор предсказывать новый отклик y′=I[y=c]y'=\mathbb{I}[y=c]y′=I[y=c]. Полученный классификатор будет определять, верно ли, что объект xxx принадлежит классу ccc или нет по правилу:

y^=sign(gc(x))\hat{y}=sign(g_c(x))y^​=sign(gc​(x))

с некоторой обученной для этого относительной дискриминантной функцией gc(x)g_c(x)gc​(x).

Тогда решающее правило для многоклассовой классификации будет определяться по правилу

y^=arg⁡max⁡cgc(x),\hat{y}=\arg\max_c g_c(x),y^​=argcmax​gc​(x),

то есть будет назначаться тот класс, за который будет голосовать классификатор этого класса с самой большой уверенностью.

Это решающее правило очень похоже на общий вид многоклассового классификатора, но тут нужно помнить, что для нахождения всех дискриминантных функций необходимо обучить CCC бинарных классификаторов на одной и той же выборке, но с разными откликами.
